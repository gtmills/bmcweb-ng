auth,rbac: enforce per-route privilege checks on all mutating handlers

Wire check_privilege() into every PATCH, POST (action/create), and DELETE
handler across the Redfish API surface.  Handlers behind auth_middleware
now enforce RBAC before executing any state-changing operation.

Privilege mapping applied:
  PRIVILEGE_PATCH      → patch_system, patch_chassis, patch_network_protocol,
                         patch_manager_ethernet_interface, patch_session_service,
                         patch_event_service
  PRIVILEGE_ACTION     → reset_system, reset_manager, clear_event_log,
                         clear_manager_bmc_log, submit_test_event,
                         delete_task, simple_update
  PRIVILEGE_CONFIGURE_USERS → patch_account_service, create_account,
                              patch_account, delete_account,
                              create_subscription, delete_subscription
  DELETE session       → allows own-session deletion (ConfigureSelf) or
                         any session deletion (ConfigureUsers)

All mutating handler signatures now accept Extension<UserSession> from
the auth middleware.  Unit tests updated to pass an administrator session
where handlers are called directly.

122 unit tests passing; zero clippy warnings.
