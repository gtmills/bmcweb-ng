websocket: implement /vm/0/0 and /nbd/0 virtual media UNIX-socket proxy

Add vm_handler (/vm/0/0) and nbd_handler (/nbd/0) WebSocket endpoints,
both backed by handle_virtual_media() — a bidirectional UNIX domain socket
proxy following the same pattern as handle_serial_console and handle_kvm.

Upstream bmcweb (features/virtual_media/vm_websocket.hpp) connects to a
UNIX socket path discovered via xyz.openbmc_project.VirtualMedia.MountPoint
DBus properties.  Our implementation connects directly to well-known paths:
  /vm/0/0  → /run/media-proxy/slot_0
  /nbd/0   → /run/media-proxy/nbd_0

Both paths match the OpenBMC nbd-proxy / obmc-usb-proxy convention.
Buffer size is 128 KiB + 16 bytes (NBD max message size per protocol spec).

Routes added to websocket_routes() router.
