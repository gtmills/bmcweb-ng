websocket,kvm: implement KVM TCP proxy to obmc-ikvm at 127.0.0.1:5900

Replace the KVM stub with a real bidirectional proxy matching upstream
bmcweb (features/kvm/kvm_websocket.hpp).

Upstream connects to TCP port 5900 on localhost where the obmc-ikvm
daemon runs a VNC/RFB server.  The bmcweb layer is a transparent proxy
— it does not speak RFB directly; it forwards raw bytes between the
WebSocket and the TCP connection.  The VNC/RFB protocol is handled by
obmc-ikvm and the browser-side noVNC client.

Implementation mirrors the serial console handler:
  - WebSocketUpgrade accepted; handle_kvm() spawned on-upgrade
  - TcpStream::connect("127.0.0.1:5900"); on failure sends WS 1011 close
  - Two tokio::spawn tasks: tcp_to_ws and ws_to_tcp
  - tokio::select! waits for either direction to finish
  - Binary and Text WS frames both forwarded as raw bytes
