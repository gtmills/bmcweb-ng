api: implement DBus REST API (/bus/, /list/, /xyz/*, /org/*)

Add src/api/dbus_rest.rs with a new dbus_rest_router() matching the
upstream bmcweb openbmc_dbus_rest.hpp feature.

Endpoints implemented:
  GET /bus/                 → lists available buses (["system"])
  GET /bus/system/          → calls org.freedesktop.DBus.ListNames and returns
                              sorted well-known service names
  GET /list/                → enumerates all objects via GetManagedObjects on
                              xyz.openbmc_project.ObjectMapper
  GET /xyz/<path>           → returns all interface properties of a DBus object
                              at /xyz/<path>, querying candidate managed-object
                              services (Inventory, State, Network, Logging, …)
  PUT /xyz/<path>           → sets a single property via set_property(); body:
                              { interface, property, value }
  GET /org/<path>           → same as /xyz/<path> for /org/* object paths
  PUT /org/<path>           → same as PUT /xyz/<path>

Routes wired into http.rs behind mandatory auth_middleware.
Module registered in src/api/mod.rs.
3 unit tests added (get_buses, get_bus_system no-dbus, get_list no-dbus).
