http: add static WebUI file serving at /ui

Mount tower-http ServeDir at /ui, serving the OpenBMC Web UI from
/usr/share/www (conventional OpenBMC installation path).  Falls back
to ./www for local development when the system directory does not exist.

ServeFile fallback to index.html is configured for SPA client-side
routing — any path under /ui/ that does not match a physical file
returns index.html so that the browser-side router can handle it.

The /ui routes are placed outside the auth middleware layer so that
browsers can load assets without a session token.  Authentication is
enforced at the API level (Redfish/DBus REST) not the asset level.
